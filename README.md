# 猎手运营中台

第一版是轻量本地工作台，先把猎手活动的运营流程跑起来：

- 首页：目标日期、来源日期、今日广告任务、异常备注
- 首页：今日待办、数据摘要、待确认广告任务、异常提醒
- 内容创作：社群文案、Push 标题/摘要、活动话术、直播/课程链接话术、素材备注
- 用户洞察：人群分组管理、高点击人群记录、低效人群记录、后台目标用户备注
- 活动策划：活动周期、每日素材链接、推广节奏、四类渠道计划、活动复盘
- 数据报表：后台广告统计入口、数据导入/手动录入、日报、周报、月报、问题素材分析
- 数据策略：App/PC 弹窗低效素材识别和同人群替换建议
- 日报增强：当前日期/对比日期自由选择、建议替换图、创新方向、社群文案分析
- 日报详细页：核心指标、成交漏斗、渠道拆分、命中人群拆分、问题素材 TOP、加入待办、复盘记录、导出 Word/Excel/PPT 占位
- 智能工作流：广告计划、T01-T04、铺设进度、异常处理、执行结果
- 后台入口矩阵：普通广告、App 弹窗、PC 弹窗、Push 的菜单位置和常用动作
- 左侧二级导航：主栏目打开总览，子栏目打开独立页面

固定规则：

- 产品/广告分组：猎手活动
- 产品别名：无
- 长期广告：无
- 普通广告、App 弹窗、PC 弹窗、Push 的来源日都等于目标日前 1 天
- 社群、直播、Push、换图策略默认只生成内容或计划，不自动发送
- 广告执行前必须先查重、人工确认、执行后回查

使用方式：

1. 双击 `启动猎手运营中台.ps1`
2. 如果本机没有可用 Python，脚本会直接打开 `index.html`
3. 页面数据会先保存在浏览器本地

刷新猎手活动后台广告数据：

```powershell
powershell -ExecutionPolicy Bypass -File scripts\update-hunter-adstat.ps1 -Date 2026-07-06
```

说明：

- 需要先在带调试端口的浏览器里打开并登录广告统计后台
- 当前猎手活动广告分组 ID：`52`
- 脚本会写入 `data/hunter-adstat-data.js`
- 页面刷新后会自动加载后台数据；也可以在 `数据报表 -> 后台广告统计入口` 点击“加载后台数据”
- 后台数据会同步驱动：首页数据摘要、日报总览、渠道拆分、人群拆分、问题素材 TOP、App/PC 策略建议、用户洞察
- 日报刷新脚本会同时拉取当前日、前一日、前前一日，用于指标对比徽标和三日趋势。
- 日报行情简述读取东方财富公开指数表现接口（上证指数、深证成指、创业板指），渲染市场情绪和指数表现；右侧 K 线图读取东方财富公开日线接口。

后台数据入口：

https://zt.jingzhuan.cn/zt#/operation/circle/sheet/operate/adstat

社群文案入口：

https://zt.jingzhuan.cn/#/operation/circle/qiwei/material

社群文案标签口径：`猎手活动`

刷新猎手活动企微素材数据：

```powershell
powershell -ExecutionPolicy Bypass -File scripts\update-hunter-community-copy.ps1 -Date 2026-07-07 -MaxPages 20
```

当前已验证：

- 统计范围：`企业微信 -> 素材管理 -> 猎手栏目`
- 分别统计并汇总标签：`主题猎手`、`策略猎手`、`AI猎手`、`猎手活动`
- 汇总后的社群文案指标会进入日报核心指标：文案当天总数、引用总数、平均引用次数
- `猎手活动` 企微素材标签 ID：`52`
- 标签素材可抓取；为避免接口 429，可用 `-MaxPages` 控制每个标签最多抓取页数
- 如果当天创建文案为 0，页面会显示当天文案数为 0，不影响广告日报
